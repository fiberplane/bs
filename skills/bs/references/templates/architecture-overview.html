API Gateway Architecture

Edge-first request routing with auth, rate limiting, and observability

RFC v1.0 March 2026

System Overview

The API gateway sits at the edge, handling authentication, rate limiting, and request routing before traffic reaches backend services. Built on Cloudflare Workers for global low-latency.

Core Components

Edge Worker

Entry point for all requests. Handles TLS termination, geoIP enrichment, and request normalization. Deployed to 300+ edge locations.

Auth Middleware

Validates session cookies and Bearer tokens. Resolves user identity from Better Auth sessions. Attaches user_id to request context.

Rate Limiter

Token bucket algorithm per user/IP. Backed by Durable Objects for distributed state. Returns 429 with Retry-After header.

Request Router

Hono-based routing tree. Matches request path to handler. Supports middleware chains, wildcard routes, and WebSocket upgrades.
